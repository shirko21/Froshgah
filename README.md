# FlexiShop Starter — قالب فروشگاهی چندمنظوره

یک پروژه خام و قابل استفاده مجدد برای ساخت فروشگاه‌های مختلف. تمام محتوای اصلی فروشگاه از پنل مدیریت تغییر می‌کند: نام و لوگو، رنگ‌ها، صفحه اصلی، محصولات، دسته‌بندی‌ها، بنرها، صفحات، سفارش‌ها، روش ارسال، روش پرداخت و کدهای تخفیف.

## امکانات

- فروشگاه واکنش‌گرا و فارسی/RTL
- جستجو و فیلتر محصولات
- سبد خرید و ثبت سفارش مهمان
- موجودی، قیمت قبل/بعد، محصول ویژه و گالری تصاویر
- کد تخفیف درصدی یا مبلغ ثابت
- روش‌های ارسال و پرداخت قابل مدیریت
- صفحات درباره ما، قوانین و تماس با ما
- پنل ادمین با ورود امن، داشبورد و مدیریت کامل
- آپلود تصویر با محدودیت نوع و حجم
- PostgreSQL، مهاجرت خودکار دیتابیس و داده نمونه
- Docker و اجرای توسعه‌ای بدون Docker
- API آماده اتصال به درگاه پرداخت واقعی

## اجرای سریع با Docker

1. فایل `.env.example` را به `.env` کپی کنید.
2. حتماً `JWT_SECRET` و `ADMIN_PASSWORD` را تغییر دهید.
3. اجرا:

```bash
npm run docker:up
```

یا مستقیماً:

```bash
docker compose up -d --build
```

سایت: `http://localhost:5000`
پنل: `http://localhost:5000/admin`

## اجرای توسعه‌ای

نیازمندی‌ها: Node.js 20+ و PostgreSQL.

```bash
cp .env.example .env
npm install
npm run db:migrate
npm run db:seed
npm run dev
```

فرانت‌اند: `http://localhost:5173`
API: `http://localhost:5000/api/health`

## ورود اولیه

ادمین در اولین اجرای بک‌اند از `ADMIN_EMAIL` و `ADMIN_PASSWORD` ساخته می‌شود. پس از ورود، رمز را در بخش «مدیران» تغییر دهید.

## ساخت یک فروشگاه جدید برای مشتری دیگر

برای هر مشتری یک کپی مستقل از پروژه بسازید و دیتابیس/فایل `.env` جداگانه بدهید. سپس از پنل مدیریت:

1. تنظیمات فروشگاه و رنگ‌ها را تغییر دهید.
2. دسته‌بندی و محصولات نمونه را حذف یا ویرایش کنید.
3. بنرها، صفحات، ارسال و پرداخت را تنظیم کنید.
4. دامنه و درگاه پرداخت اختصاصی مشتری را متصل کنید.

## اتصال درگاه پرداخت

قالب خام، پرداخت‌های `cod`، `bank_transfer` و `manual` را پشتیبانی می‌کند. برای پرداخت آنلاین، یک provider جدید در `apps/api/src/services/payment-providers.js` اضافه کنید و webhook رسمی همان درگاه را پیاده‌سازی کنید. هیچ کلید پرداختی را داخل فرانت‌اند قرار ندهید.

## نکات امنیتی تولید

- رمز و JWT_SECRET پیش‌فرض را تغییر دهید.
- HTTPS فعال کنید.
- `WEB_ORIGIN` را فقط روی دامنه اصلی تنظیم کنید.
- از دیتابیس و پوشه uploads نسخه پشتیبان بگیرید.
- برای درگاه واقعی، webhook و امضای تراکنش را سمت سرور بررسی کنید.

## ساختار

```text
apps/api   Express + PostgreSQL
apps/web   React + Vite
```

## محدودیت نسخه خام

این نسخه سیستم حساب کاربری مشتری، چندفروشندگی و صدور فاکتور رسمی مالیاتی ندارد. ساختار پروژه برای افزودن این امکانات آماده است، اما باید متناسب با قوانین و نیاز هر کسب‌وکار توسعه داده شود.
